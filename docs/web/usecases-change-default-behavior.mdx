Many of the behaviors of Geocortex Web's built in [components](concepts-components-services.mdx) have behaviours which can be configured through [app config](concepts-app-config.mdx).  

The follow example overrides the default behaviour for a map click by modifying the `onClick` property in the app config for the map component.

<Tabs
  defaultValue="config"
  values={[
    { label: "App Config", value: "config" },
    { label: "Layout", value: "layout" },
  ]}>
<TabItem value="config">

```json
{
    "$schema": "..\\..\\ViewerFramework\\app-config\\mobile\\mobile-app-config.schema.json",
    "schemaVersion": "1.0",
    "items": [
        ...
        {
            "$type": "map-extension",
            "id": "map1",
            "onClick": [ "tasks.identify", "highlights.pulse", "results.display" ]
        },
        ...
    ]
}
```

</TabItem>
<TabItem value="layout">


```xml
<?xml version="1.0" encoding="UTF-8"?>
<layout xmlns="https://geocortex.com/layout/v1" xmlns:custom="your.custom.namespace" xmlns:web="https://geocortex.com/layout/web/v1">
    <map config="map1"/>
</layout>
```

</TabItem>
</Tabs>

The `onClick` property for the `<map>` has been configured to run a series of [commands and operations](concepts-commands-operations.mdx). These commands and operations will be executed on a map click instead of the default map click behaviour. By using the [`workflow.run` command](api-commands-operations.mdx#workflowrun), you can even execute a [workflow](https://docs.geocortex.com/workflow/latest/help/Default.htm#wf5/help/geocortex-workflow.htm#Introduction_to_Geocortex_Workflow%3FTocPath%3D%250A%2520%2520%2520%2520%2520%2520%2520%2520%2520%2520%2520%2520Introduction%2520to%2520Geocortex%2520Workflow%7C_____0) with [completely configurable behaviour](usecases-display-custom-form.mdx). 

```json
{
    "$schema": "..\\..\\ViewerFramework\\app-config\\mobile\\mobile-app-config.schema.json",
    "schemaVersion": "1.0",
    "items": [
        ...
        {
            "$type": "workflow",
            "id": "customClickWorkflow",
            "title": "Custom Map Click Workflow",
            "portalItem": "<your-workflow-id-here>"
        },
        {
            "$type": "map-extension",
            "id": "map1",
            "onClick": [
                {
                    "name": "workflow.run",
                    "arguments": {
                        "id": "customClickWorkflow"
                    }
                }
            ]
        },
        ...
    ]
}
```

## Discovering Configurable Properties

You can view the configurable properties for a given component in the API documentation. For example, [here's the documentation](api-components.mdx#map) for the `<map>` component with the `onClick` property.

By using commands and operations to customize default behaviour and using [layout principles](concepts-layout.mdx) to modify the UI, you can change deeply customize a Geocortex Web Application without having to write custom code.
